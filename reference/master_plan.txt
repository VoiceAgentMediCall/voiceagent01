MediCall AI
MVP Master Plan
AI Voice Agent for Elderly Medication Adherence (India)
Based on research doc: "AI Medication Reminder for Elderly"

Context
Nearly 48% of Indian chronic-disease patients are non-adherent to medication. The primary caregiver (adult child, 28-45, in a different city) cannot physically verify daily intake. The product solves this with an outbound AI voice call to the elderly parent at scheduled times — no smartphone app required on the parent's side. The caregiver gets a web dashboard to set schedules and track adherence.

MVP Goal: Validate the core loop — schedule → AI call → confirmation logged → caregiver notified — with 50 beta user dyads (caregiver + parent pairs) within 1 month of heavy build.

1.2 Feature Set — Three Phases
Phase A: Core Loop (MVP — Month 1)
These are the only things you ship to 50 users.

A1 — Caregiver Onboarding & Auth
Email + password signup with OTP email verification
After login: create a "dependent profile" — parent's name, phone number, preferred language (Hindi / Tamil / Bengali / Telugu / Malayalam)
DPDP consent flow: system sends an OTP SMS to the parent's phone; caregiver inputs it to bind the accounts (this is the legal proxy consent step)
Session management via JWT (Supabase Auth handles this natively)
A2 — Medication Schedule Setup
Manual entry mode: drug name, dosage, frequency, time(s) of day, duration, relation to meals
OCR mode: upload a photo of a prescription → Veryfi Medical OCR API extracts a JSON draft → caregiver reviews + edits in a side-by-side view (original image left, editable form right) → approves
Drug validation layer: backend cross-checks extracted drug names against Indian medicine DB (Apollo Pharmacy dataset or 1mg API) and flags unrecognized terms with suggestions
Weekly calendar view: shows each medication slot color-coded (scheduled / completed / missed / pending)

A3 — AI Voice Call Engine
Cron trigger: Upstash QStash fires a durable job at scheduled medication time → backend calls Vapi.ai outbound call API
Vapi orchestration: STT via Sarvam Saaras (vernacular), LLM for intent, TTS via Sarvam Bulbul (vernacular)
Telephony: Exotel SIP trunk (India-local routing, <50ms media latency, TRAI-compliant caller ID)
Call script flow (see Section 1.3 for full dialogue tree)
On call end: Vapi webhook fires to backend → backend logs outcome (confirmed / denied / missed / deferred) to Supabase
A4 — Three-Strike Escalation
If call is unanswered: retry after 15 min, then 30 min (3 total attempts)
After 3 strikes: halt automated calls for that event, fire escalation alert to caregiver via WhatsApp (Twixor or Plivo WA Business API) AND in-app push notification
Alert payload includes: drug name, scheduled time, reason for escalation
If patient verbally reports a side effect (nausea, dizziness): AI agent empathetically acknowledges, does NOT suggest any action, immediately triggers high-priority escalation to caregiver
A5 — Caregiver Dashboard
Home: adherence streak calendar (green = confirmed, red = missed, grey = pending)
Call log view: every call with timestamp, outcome, and text transcript
Medication list: active medications with edit/pause/delete
Notification settings: configure escalation channel preference (WhatsApp / email)

A6 — Admin Dashboard (operator)
Total users (caregiver-patient dyads)
Daily/weekly calls made vs answered vs missed
Task Completion Rate (TCR), Call Abandonment Rate, average call duration
Transcript audit view: random sample of 5 calls/day for hallucination check
API spend tracker per component (Vapi, Sarvam, Exotel, Veryfi)
Escalation events log

1.3 Voice Call Dialogue Structure
Happy Path (patient picks up, confirms):
AI:  "Namaste [Parent name], main [AI name] bol raha hoon. Aapka [Drug name],
      [dosage], lene ka samay ho gaya hai. Kya aapne abhi tak liya hai?"
PATIENT: "Haan, le liya" (or code-mixed: "Yes, le liya")
AI:  "Bahut achha! Main note kar leta hoon. Apna khayal rakhiyega."
→ Call ends. Backend logs: CONFIRMED.

Deferral Path:
PATIENT: "10 minute mein lo, abhi khaana kha raha hoon"
AI:  "Bilkul, main 15 minute baad call karta hoon."
→ Call ends. Backend queues a new QStash job for T+15min.

Side-Effect Path:
PATIENT: "Woh dawai lene ke baad chakkar aata hai"
AI:  "Mujhe bahut dukh hua yeh sun ke. Kripya abhi yeh dawai mat lijiye aur
      apne doctor se zaroor baat karein. Main aapke bete/beti ko turant
      inform kar deta hoon."
→ Call ends. Backend triggers IMMEDIATE escalation. Logs: ESCALATED-SIDE-EFFECT.

No-Answer Path:
→ Call rings, no answer / voicemail detected
→ Backend logs: ATTEMPT_1_FAILED
→ QStash queues retry after 15 min
→ After 3 failures: logs ESCALATED-NO-ANSWER, fires WhatsApp alert to caregiver

Strict Guardrails (hardcoded in LLM system prompt):
NEVER alter dosage, timing, or medication names
NEVER diagnose symptoms
NEVER recommend stopping or starting any medication
ALWAYS suggest consulting a doctor for medical concerns
NEVER recall information from prior calls (stateless per conversation)

1.4 Evaluation Metrics (Production Targets)

Admin monitoring routine (daily, 15 min):
Read Vapi dashboard for latency P95
Sample 5 random transcripts for hallucination check
Review all escalation events from prior day
Check API spend vs budget

2.4 Critical Engineering Decisions
Why Render Starter ($7/mo) not free tier:
Free Render sleeps after 15 min inactivity. Vapi webhooks during a live call would hit a sleeping server → 30-60 second cold start → patient hears dead silence → call abandoned. Non-negotiable paid upgrade.
Why Supabase Pro ($25/mo) not free tier:
Medical data = Sensitive Personal Data (SPD) under DPDP Act. Free tier has no PITR backups. A DB corruption or accidental delete = regulatory violation + patient harm. Non-negotiable paid upgrade.
Why Upstash QStash not basic cron:
Basic cron is fire-and-forget. If Vapi API is momentarily down, the job silently fails — patient misses reminder with no retry. QStash has built-in exponential backoff: retries at 5s → 15s → 1min. Also provides observability dashboard.
Why Exotel not Twilio:
Twilio India routing adds 200-800ms network latency (transoceanic hops). Exotel uses local Indian infra → <50ms media latency. Also 30-40% cheaper per minute for Indian numbers.
Why Sarvam not global models:
GPT-4o Whisper fails on heavy regional accents and code-mixing (Hindi-English mixed sentences). Sarvam Saaras is purpose-built for this exact demographic.
Why Veryfi not custom ML (for MVP):
Open-source MLLMs (Qwen-VL) achieve ~82% accuracy on Indian handwritten Rx — insufficient for production healthcare. Veryfi achieves 99%+. The plan is to build a proprietary dataset during MVP, then fine-tune and replace Veryfi in Phase C.
2.5 DPDP Act 2023/2025 Compliance Checklist
[ ]  OTP-based proxy consent flow for parent during onboarding (with timestamp audit log)
[ ]  Data Processing Agreements (DPAs) signed with Vapi, Sarvam, Veryfi, Exotel prohibiting training data use
[ ]  Row Level Security on all Supabase tables
[ ]  AES-256 encryption at rest (Supabase Pro provides this)
[ ]  TLS 1.3 in transit everywhere
[ ]  Auto-purge of raw audio recordings and prescription images after 30 days (Supabase lifecycle rule)
[ ]  User-facing 'Right to Erasure' button → cascading deletion across all tables + storage
[ ]  Breach notification workflow: internal alerting + 72-hour window to notify Data Protection Board
[ ]  Privacy Policy in plain language Hindi + English
[ ]  Data minimization: only store what's needed (no raw audio after transcript is generated)

Section 5 — Pre-Build Research Checklist
Before writing a single line of code, complete these:

5.1 Vendor Validation (Week 0 — Before Month 1)
[ ]  Vapi.ai: Create account, make a test outbound call to an Indian mobile number. Verify actual latency from India. Apply for startup credits.
[ ]  Sarvam AI: Get API keys, test Saaras STT with a real Hindi voice recording. Test Bulbul TTS. Confirm custom provider webhook works in Vapi.
[ ]  Exotel: Call their sales team. Get a test Exophone number. Understand TRAI DND compliance — do they scrub against DND registry automatically? Get pricing in writing.
[ ]  Veryfi: Upload 5 sample Indian doctor prescriptions (handwritten). Evaluate JSON output accuracy. If <90% on your samples, evaluate Koncile as fallback.
[ ]  Twixor/Plivo WA API: Confirm WhatsApp Business account creation timeline (Facebook verification takes 3-5 days — start this on Day 1).

5.2 Legal Research
[ ]  Confirm whether an elderly patient with full cognitive capacity needs a separate written consent form beyond the OTP flow under DPDP 2025 rules (consult a legal advisor — 1 hour, ~₹3,000)
[ ]  Understand TRAI regulations around outbound automated calls — are there restrictions on AI-initiated calls to Indian mobiles? (Exotel should advise on this — it is a real risk)
[ ]  Check if DPDP 'Significant Data Fiduciary' designation applies (likely not at MVP scale, but know the threshold)

5.3 User Research (Before Building UI)
[ ]  Do 5 discovery calls with caregivers: What does their current reminder workflow look like? WhatsApp? Manual calls? What times do their parents typically take medicines?
[ ]  Do 2-3 calls with elderly parents (or family members who are 65+): How do they feel about receiving calls from an unknown/AI number? What language + tone works? What makes them hang up?
[ ]  Validate: Is 3 medicines/day a realistic average? (This drives your cost model significantly)

5.4 Technical Spikes (Before Committing to Architecture)
[ ]  Latency stack test: Set up Vapi + Sarvam + Exotel in isolation, make one real call, measure actual P95 latency end-to-end. If >1000ms, investigate bottleneck before building.
[ ]  Indian handwriting OCR test: Get 10 real doctor prescriptions (blurred for privacy), run through Veryfi. Pass rate must be >90% before committing to Veryfi.
[ ]  Hindi code-mixing NLU test: Test Sarvam STT on sentences like 'Haan, le liya' and 'Abhi busy hoon, 10 minutes mein call karo.' Verify it transcribes accurately.
